FlowJo SweetSpot Web

Load a FlowJo-exported .csv or .xlsx, tune the panel parameters, and run the sweet-spot analysis directly in the browser.

Inputs

Population base is the parent gate used in the FlowJo export, for example CHO/Singlets.

Analysis mode controls how the best condition is selected after safety filtering.

Detector max, clipping k, and safe fraction define the saturation rule:

UpperEnvelope = Median_pos + k * rSD_pos

Empty-gate threshold tells the app when a neighbor gate should be treated as biologically empty instead of problematic missing data.

Global SI / Ws thresholds are fallback values. In rule_based_per_dye, the dye table below can override them.

Dye configuration lets you edit the target fluorescence channel and the per-dye selection logic.

Metric trends

Stain Index shows separation from the unstained control. Higher is better.

Ws is the weighted spill / spread penalty into the other channels. Lower is better.

Quality Score is a descriptive combined score, but the final decision does not rely only on it.

Upper envelope shows detector pressure using Median + k * rSD. Staying lower is safer.

Color and symbol are assigned by clone, so the same clone keeps the same visual identity across figures.

Selection criteria

Each point is one tested condition.

Y axis: Stain Index, so higher means stronger separation from ST.

X axis: Ws on a log scale, so moving left means less spill / intrusion.

The recommended region is high enough in SI, left enough in Ws, and outside clipping risk.
